# v15 SK서린빌딩 보유세 계산 로직

## 1. 계산 원칙

계산 엔진은 모든 금액 연산에 `Decimal`을 사용하고, 공식 검증 상태와 법적 근거 URL이 있는 세법 규칙표만 적용합니다. 필수 입력, 법적 분류 또는 출처가 부족하면 세액을 만들지 않습니다.

공개 계산범위는 SK리츠·SK서린빌딩·2026년으로 고정합니다.

## 2. 토지 관련 세액

```text
토지 시가표준액 = 개별공시지가 × 과세면적 × 소유지분
재산세 과세표준 = 토지 시가표준액 × 공정시장가액비율 70%
토지분 재산세 = 재산세 과세표준 × 분리과세 세율 0.2%
토지분 재산세 도시지역분 = 재산세 과세표준 × 0.14%
토지분 지방교육세 = 토지분 재산세 × 20%
```

토지 시가표준액은 `63,950,000원/㎡ × 5,773.5㎡ × 100% = 369,215,325,000원`입니다.

## 3. 건축물 관련 세액

```text
건축물 재산세 과세표준 = 건축물 시가표준액 × 공정시장가액비율 70%
건축물분 재산세 = 재산세 과세표준 × 0.25%
건축물분 재산세 도시지역분 = 재산세 과세표준 × 0.14%
건축물분 지방교육세 = 건축물분 재산세 × 20%
```

공식 건축물 시가표준액은 `40,221,752,104원`입니다. 투자부동산 장부가액이나 감정가로 대체하지 않습니다.

## 4. 소방분 지역자원시설세

```text
기본 누진세액 = 49,100원 + (건축물 시가표준액 - 64,000,000원) × 0.12%
소방분 지역자원시설세 = 기본 누진세액 × 300%
```

업무시설·지상 36층이라는 공개자료와 지방세법 시행령 제138조의 법정 요건을 연결하여 대형 화재위험 건축물 300% 배율을 적용했습니다. 실제 과세내역서의 위험유형 코드는 미확인이고 고지세액 대사는 완료되지 않았습니다.

## 5. 토지 분리과세와 종합부동산세

공개 사실관계와 현행 법령을 연결하면 SK서린빌딩 토지는 공모부동산투자회사 목적사업용 토지의 분리과세 적용요건을 충족하는 것으로 분석됩니다. 이 판단은 토지에만 적용합니다.

분리과세 토지는 종합부동산세법 제11조의 종합합산·별도합산 토지 과세대상에 포함되지 않으므로 `토지분 종합부동산세: 과세대상 제외`로 표시합니다. 선행 종합부동산세가 없으므로 `종합부동산세분 농어촌특별세: 해당 없음`으로 표시합니다. 계산 데이터의 0원은 이 법적 상태를 기계적으로 표현한 보조값입니다.

## 6. 세목별 끝수 처리

각 세목은 세율·배율을 적용한 산출세액을 보존한 뒤 다음 함수를 적용합니다.

```python
def truncate_to_ten_won(amount: Decimal) -> Decimal:
    if amount < 0:
        raise ValueError
    return (amount // Decimal("10")) * Decimal("10")
```

적용 순서는 `과세기초자료 -> 시가표준액 -> 과세표준 -> 세율·배율 -> 세목별 산출세액 -> 세목별 끝수 처리 -> 처리 후 금액 합산`입니다. 전체 합계를 한 번만 처리하지 않습니다. 시가표준액, 과세표준, 세율, 배율과 과세대상 제외·해당 없음 행에는 적용하지 않습니다.

| 세목 | 끝수 처리 전 | 끝수 처리 후 |
|---|---:|---:|
| 토지분 재산세 | 516,901,455 | 516,901,450 |
| 토지분 재산세 도시지역분 | 361,831,018.5 | 361,831,010 |
| 토지분 지방교육세 | 103,380,291 | 103,380,290 |
| 건축물분 재산세 | 70,388,066.182 | 70,388,060 |
| 건축물분 재산세 도시지역분 | 39,417,317.06192 | 39,417,310 |
| 건축물분 지방교육세 | 14,077,613.2364 | 14,077,610 |
| 소방분 지역자원시설세 | 144,715,207.5744 | 144,715,200 |
| **합계** | **1,250,710,968.55472** | **1,250,710,930** |

처리 차이는 `38.55472원`입니다. 지방회계법 제55조는 선택 규정이고 시행령 제67조의 예외가 있으므로 실제 과세관청의 끝수 처리 방식은 과세내역서로 별도 확인해야 합니다.

## 7. 보유세 민감도 분석

민감도 분석은 같은 계산 엔진을 다시 호출하고 세율이나 산식을 UI에 중복 구현하지 않습니다.

| 민감도 분석 | 변동률 | 끝수 처리 전 합계 | 끝수 처리 후 합계 | 기준 대비 증감액 |
|---|---:|---:|---:|---:|
| 기준 | 0% | 1,250,710,968.55472 | 1,250,710,930 | 0 |
| 공시가격·시가표준액 5% 상승 | +5% | 1,313,250,671.982456 | 1,313,250,630 | 62,539,700 |
| 공시가격·시가표준액 10% 상승 | +10% | 1,375,790,375.410192 | 1,375,790,350 | 125,079,420 |

이는 개별공시지가와 건축물 시가표준액의 기계적 변화 분석이며 미래 결정세액 또는 과세관청 고지세액의 예측이 아닙니다.

## 8. 회귀 통제

- `calculated_tax`는 끝수 처리 후 재계산액과 일치해야 합니다.
- 끝수 처리 전·후 금액과 차이는 별도 컬럼으로 보존해야 합니다.
- 기준 시나리오와 세목별 합계는 위 표의 정확한 `Decimal` 값과 일치해야 합니다.
- 실제 고지서 미확인 상태에서 `verified_notice`를 생성하지 않습니다.
- 장부가액 또는 Peer 비율을 대체값으로 사용하지 않습니다.
