# v15 검증 정책

## 품질 차원

- **완전성**: 전체 상장리츠, 자산, 필지, 건축물과 납세의무자 Coverage
- **유일성**: Stock Code, Asset ID, Parcel ID, Building ID와 계산 Grain 중복
- **유효성**: 19자리 PNU, 소유지분 범위, 허용 상태와 숫자 범위
- **일관성**: 자산-필지-건축물-납세의무자 관계와 과세구분
- **무결성**: 고아 Key, 납세의무자별 전국 합산과 Source URL
- **적시성**: 과세연도, 원천 문서일, 조회일과 법령 시행기간

## 핵심 통제

1. 공식 Source URL이 없는 숫자는 계산 차단
2. 자산별 납세의무자 미확인 또는 복수 후보 차단
3. PNU·면적·지분·개별공시지가 중 하나라도 없으면 토지 계산 차단
4. 건축물 시가표준액이 없으면 건축물 재산세·소방분 계산 차단
5. 분리과세 판정이 불명확하면 토지 재산세와 종부세 결론 차단
6. 도시지역분 고시·조례와 소방분 위험유형 미확인 시 해당 세목 차단
7. 전국 합산·재산세 공제액이 없으면 종부세 계산 차단
8. 검토 필요 상태에 `calculated_tax`가 존재하면 Critical 오류
9. 실제 고지서가 없으면 Reconciliation을 Open 상태로 유지
10. 세금과공과 전체를 보유세 Ground Truth로 사용하지 않음

## 자동 생성 요청자료

검증 실패는 등기부등본, 신탁원부, 토지·건축물대장, PNU 목록, 건축물 시가표준액, 분리과세 판정자료, 도시지역분·소방분 자료와 실제 고지서 요청으로 연결됩니다.

## 테스트

`tests/test_tax_v15.py`는 필지·지분 계산, 재산세·부가세목·종부세, 신탁 납세의무, 분리과세 차단, 장부가액·Peer fallback 금지, Source 없는 숫자 차단과 Golden Test를 검증합니다.
