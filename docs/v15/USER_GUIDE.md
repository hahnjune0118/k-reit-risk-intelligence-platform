# v15 Tax 사용자 가이드

## 시작

Streamlit에서 **Tax: 보유세 분석**을 선택한 뒤 분석대상 리츠와 기준연도를 선택합니다. 자산, 납세의무자, 직접·간접 보유구조와 계산상태 필터를 사용할 수 있습니다.

## 화면 읽는 순서

1. 분석대상 리츠와 기준연도
2. 자산 및 납세의무자 구조
3. 공모부동산투자회사 법적 요건
4. 분리과세 적용 요건
5. 과세기준일 및 납세의무자
6. 자산별 주소와 PNU
7. 토지 시가표준액 도출
8. 토지 재산세 계산
9. 건축물 시가표준액 도출
10. 건축물 재산세 계산
11. 도시지역분·지방교육세
12. 소방분 지역자원시설세
13. 종합부동산세·농어촌특별세
14. 총 보유세, 검증 결과 및 요청자료

각 단계는 결론, 법규, 요건, 공식, 실제 입력, 결과, 출처와 한계를 순서대로 제시합니다.

## 상태 해석

- **고지서 확인**: 실제 과세자료와 대사된 상태
- **공식자료 계산**: 공식 입력값과 검증 규칙으로 계산한 상태
- **공식자료 일부**: 일부 근거만 확보된 상태
- **수동 검토**: 법적 판단 또는 전문 검토가 필요한 상태
- **데이터 부족**: 필수자료 부족으로 계산하지 않은 상태
- **해당 없음**: 검증된 분류상 세목 비적용

`데이터 부족`은 0원을 의미하지 않습니다.

## 다운로드

- 계산내역 CSV
- Assets·Parcels·Buildings·Taxpayers·Validation·Requests를 포함한 Excel 검토팩
- 17개 섹션 Markdown Tax Review Memo
- 브라우저에서 열 수 있는 HTML 검토문서

## 재현 가능한 실행

```powershell
py -m scripts.v15.run_pipeline --tax-year 2026 --all-reits --no-resume
py -m pytest -q
py -m streamlit run app.py
```

공식 원천을 다시 조회하려면 `--refresh-sources`, 네트워크 없이 Snapshot만 검증하려면 `--offline`을 사용합니다.

## 주의

이 화면은 초기 Tax Screening 결과입니다. 실제 신고·납부에는 자산별 과세내역서, 시가표준액, 소유·신탁구조, 조례, 감면, 세부담상한과 고지세액 검토가 추가로 필요합니다.
