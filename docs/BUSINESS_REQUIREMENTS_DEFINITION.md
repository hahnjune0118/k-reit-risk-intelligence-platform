# Business Requirements Definition

## 1. 목적과 범위

리츠 보유세 초기 검토를 결론 중심의 통제된 업무 흐름으로 전환합니다. 범위는 공개·공식자료 식별, 자산·필지·납세의무자 구조화, 세목별 재계산, 민감도, 검증상태, 이슈·요청자료, 계산조서, Source Lineage와 결과 내려받기입니다.

## 2. 주요 사용자

- 보유세 계산결과와 미해결 이슈를 검토하는 Tax 실무자
- 근거, 검증상태와 추가 요청자료를 승인하는 검토 책임자
- 데이터 계약, 규칙엔진과 회귀검증을 관리하는 운영 담당자
- 공개 포트폴리오에서 Data·Automation 기반 업무개선을 평가하는 검토자

## 3. As-Is 업무분석

자료 검색 → 주소·소유관계 수작업 확인 → 공식가액 별도 조회 → Excel 계산 → 근거 재확인 → 이슈·요청자료·메모 작성

핵심 문제는 자료 분산, 반복 확인, 계산과 근거의 분리, 중요 이슈의 후순위 노출, 공식자료 부족 시 임의 추정 위험입니다.

## 4. 기능요건

| ID | 기능요건 | 검수 기준 |
|---|---|---|
| FR-01 | 기준 사례 식별 | 회사, 자산, 납세의무자와 기준연도를 고유 ID로 연결한다. |
| FR-02 | 공식 입력근거 표시 | 공시가격·시가표준액 등 필수 입력근거 Coverage를 표시한다. |
| FR-03 | 세목별 재계산 | 검증된 Tax Rule Master와 기존 Decimal 계산엔진을 사용한다. |
| FR-04 | 결론 Dashboard | 재계산액, 고지서 Coverage, P0/P1와 계산 세목을 첫 탭에 표시한다. |
| FR-05 | 민감도 분석 | Base, +5%, +10%와 사용자 설정 결과를 비교한다. |
| FR-06 | 이슈·요청자료 | 우선순위, 이슈, 필요 증빙, 예상 영향과 다음 조치를 통합한다. |
| FR-07 | 계산조서 | 입력값·과세표준, 적용률·세율, 재계산액, 근거상태와 대사상태를 표시한다. |
| FR-08 | 근거 추적 | Evidence Matrix, Source Lineage, Reconciliation과 법적 근거를 확인한다. |
| FR-09 | 결과 내려받기 | CSV·Excel·Markdown·HTML 출력을 제공한다. |

## 5. 데이터요건

- 자산, 필지, 건축물, 납세의무자와 Tax Rule Master를 분리 관리합니다.
- 출처 URL, 문서명, 기준일, Source Type, 검증상태와 회수일을 보존합니다.
- 공시값, 공식 입력값, 계산값, 실제 고지값과 미확인 값을 구분합니다.
- 자산·납세의무자·필지·건축물 참조키의 무결성을 검증합니다.
- 금액 계산은 기존 Decimal 기반 엔진과 검증된 CSV Snapshot을 유지합니다.

## 6. 검증 통제요건

- 기준 재계산액과 +5%·+10% 시나리오 불변값 회귀테스트
- 공식 입력근거 5/5, 고지서 Coverage 0%, P0/P1 3/3, 세목 9개 검증
- 실제 고지서 확인 전 `verified_notice` 상태 생성 금지
- 필수 근거가 없는 계산에 대한 Fail-closed 처리
- 계산결과에서 Source Lineage와 적용 법령으로 이동 가능한 추적성
- 공개 화면과 다운로드에 미해결 이슈와 제한사항 유지

## 7. 비기능요건

| 항목 | 요건 |
|---|---|
| 재현성 | 같은 입력과 규칙 버전은 같은 결과를 생성해야 한다. |
| 추적성 | 결과에서 입력근거, 계산상태와 적용 규칙을 확인할 수 있어야 한다. |
| 보안 | 인증정보와 비공개 등기·신탁 원문을 공개 화면과 출력에 노출하지 않는다. |
| 가독성 | 결론과 후속조치를 먼저 표시하고 세부 계산은 단계적으로 공개한다. |
| 반응형 | 데스크톱과 좁은 화면에서 KPI, 차트와 탭을 읽을 수 있어야 한다. |
| 성능 | 기존 Snapshot과 캐시를 재사용하고 불필요한 외부 호출을 추가하지 않는다. |

## 8. 예외처리

- 기준 불변값이 달라지면 Tax 화면 렌더링을 차단하고 회귀 원인을 조사합니다.
- 필수 공식 입력근거가 없으면 `data_insufficient` 또는 검토 필요 상태를 반환합니다.
- 실제 고지액이 없으면 재계산액과 결정세액을 동일하게 표시하지 않습니다.
- Excel 출력 패키지를 사용할 수 없으면 다른 출력은 유지하고 Excel 버튼만 비활성화합니다.
- 빈 Evidence나 Source 표는 임의 행을 만들지 않고 데이터 부족 상태를 표시합니다.

## 9. 전문가 검토 경계

- 과세기준일 소유·신탁관계와 납세의무자 판정
- 실제 고지서상 과세구분 및 세목별 세액 대사
- 감면, 세부담상한, 법정 절사와 지방자치단체 조정
- 종합부동산세 제외의 실제 적용과 최종 세법 판단
- 신고·납부 또는 외부 의견에 대한 최종 승인

## 10. 검수기준, 제외범위와 향후 개선

검수 시 네 Tax 탭, 불변값, 차트·표 렌더링, 다운로드, 금지 문구, 컴파일, 전체 테스트, 정적검사와 패키지 정합성을 확인합니다.

제외범위는 신규 세법 로직, 원천 데이터 변경, 신규 API, 실제 고지세액 생성, 다른 자산의 확정 계산, Deals·KRX 기능, Power BI 수정과 최종 세법 판단 자동화입니다.

향후 개선은 실제 고지서 대사, 증빙 승인 이력, 검토자 Sign-off와 동일 Evidence 수준을 충족한 자산으로의 제한적 확대를 대상으로 합니다.
