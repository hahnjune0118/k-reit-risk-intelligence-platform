# AX Advisory Case Brief

## 1. Executive Summary

K-REIT Risk Intelligence Platform은 분산된 리츠 공시, 세법, 공시가격과 재무자료를 연결하여 검토대상 식별부터 계산, 검증, 쟁점과 요청자료, 검토메모까지 하나의 통제된 Workflow로 구조화한 AX 적용 사례입니다. v15.1.0은 기존 Tax·Assurance 계산을 유지하면서 업무 문제, To-Be Process와 역할분담을 명확히 설명합니다.

## 2. 업무 배경

리츠 검토에는 DART 공시, IR 자료, 주소·PNU, 공시가격, 소유·신탁관계와 세법 근거가 함께 필요합니다. 자료 형식과 보관 위치가 달라 동일 정보를 반복 확인하고 계산 근거와 검토 문서를 별도로 관리하는 부담이 발생합니다.

## 3. 고객 Pain Point

- 공시·IR·세법·공시가격 자료의 분산
- 주소·소유구조·과세정보의 반복 확인
- Excel 계산과 근거자료의 분리
- 담당자별 검토절차와 문서 양식의 차이
- 공식자료가 부족한 경우 추정값 사용 위험

## 4. As-Is Process

공시 PDF 검색 → 주소·소유관계 수작업 확인 → 공시가격 별도 조회 → Excel 세액 계산 → 근거 재확인 → 쟁점·요청자료·메모 수작업 작성

이 방식은 입력 근거, 계산 결과와 검토 문서의 연결을 개별 담당자의 작업 방식에 의존하게 만듭니다.

## 5. To-Be Process

공식자료 수집 → 자산·필지·납세의무자 구조화 → 과세기초자료 검증 → 법정 산식 계산 → 검증통제 → 주요 세무쟁점 → 추가 요청자료 → 검토메모 → 전문가 승인

To-Be Process는 모든 판단을 자동화하는 것이 아니라 반복 업무를 표준화하고 전문가 검토가 필요한 지점을 명시하는 데 목적이 있습니다.

## 6. AI·Data·Automation 적용

| 영역 | 적용 내용 |
|---|---|
| Data | 공시·주소·PNU·시가표준액·납세의무자 구조화 |
| Automation | 세액 계산, 끝수 처리, 검증, 민감도 분석, 문서 출력 |
| AI 지원영역 | 비정형 문서 구조화, 주요 쟁점 요약, 요청자료와 검토메모 초안 지원을 위한 확장 설계 |
| Control Harness | Source Grounding, Schema 검증, Fail-closed, 회귀테스트, 검증상태 관리 |
| Human Review | 법적 판단, 실제 고지서 대사, 최종 승인 |

공개 Runtime의 세액 계산은 재현 가능한 규칙엔진을 사용합니다. 생성형 AI는 비정형 문서 구조화와 검토문서 작성 지원을 위한 확장계층으로 설계하며, 확정적인 세법 판단과 신고·납부 의사결정은 전문가 검토 대상으로 유지합니다.

## 7. 통제 및 Human Review

- Source Lineage로 입력값과 공식 근거를 연결합니다.
- Schema 검증과 회귀테스트로 데이터 계약과 계산값 변화를 확인합니다.
- 공식자료가 부족하면 임의 추정 대신 `data_insufficient` 또는 검토 필요 상태를 유지합니다.
- 실제 과세내역서, 과세기준일 소유·신탁관계, 감면·세부담상한과 고지세액은 전문가가 확인합니다.

## 8. 구현 결과

| 구현 증빙 | 확인 결과 |
|---|---|
| 핵심 분석대상 자산 | SK서린빌딩 |
| 공식 과세근거자료 | 16건 |
| 주요 세무쟁점 | 6건 |
| 출력형식 | CSV·Excel·Markdown·HTML |
| 검증 방식 | Fail-closed |
| 실제 고지세액 대사 | 미완료 |

현재 구현은 공식자료 기반 구조화, 법정 산식 계산, 민감도 분석, Source Lineage, 주요 쟁점, 요청자료와 검토메모 출력을 포함합니다. 측정 근거가 없는 시간절감률, 정확도 개선률 또는 ROI는 제시하지 않습니다.

## 9. 한계

- 공개자료만으로 실제 과세내역과 모든 소유·신탁관계를 확정할 수 없습니다.
- 실제 고지서가 없어 재계산액과 결정세액의 대사가 완료되지 않았습니다.
- 감면, 세부담상한, 지방자치단체 조정과 최종 세법 판단은 별도 검토가 필요합니다.
- 생성형 AI API는 공개 Runtime에 직접 연결되어 있지 않습니다.

## 10. 향후 확장

- 공식 고지서·과세내역서 확보 후 재계산액 Reconciliation
- 과세기준일 소유·신탁관계 검증 강화
- 동일한 Evidence 기준을 충족한 자산으로 적용 범위 확대
- 통제된 환경에서 비정형 문서 구조화와 검토문서 초안 지원 계층 검증

확장은 기존 Fail-closed와 전문가 승인 원칙을 유지하는 조건으로 수행합니다.
