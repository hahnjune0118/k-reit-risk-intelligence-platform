# Business Process Case Brief

## 1. Executive Summary

K-REIT Risk Intelligence Platform은 분산된 리츠 공시, 공시가격, 세법과 재무자료를 연결하여 보유세 검토의 결론, 미해결 이슈, 계산조서와 근거자료를 한 흐름에서 확인하도록 만든 데이터 기반 업무개선 사례입니다. v15.1.0은 검토자가 먼저 결론과 필요한 후속조치를 판단하고 세부 계산과 근거를 단계적으로 확인하는 Decision-First 구조를 적용합니다.

## 2. 업무 배경

리츠 보유세 검토에는 자산·필지 정보, 주소와 PNU, 공시가격·시가표준액, 소유·신탁관계, 납세의무자, 세법과 실제 고지서가 함께 필요합니다. 자료의 형식과 보관 위치가 달라 입력값 확인, 계산, 근거 추적과 요청자료 작성이 분리되기 쉽습니다.

## 3. 사용자 Pain Point

- 공시, 공식가액, 소유구조와 세법 자료가 여러 경로에 분산됨
- 주소·PNU·납세의무자와 과세구분을 반복 확인해야 함
- 계산 결과와 원천 근거가 별도 파일에 존재함
- 중요한 미해결 이슈와 후속 요청자료가 긴 계산 절차 뒤에 나타남
- 공식자료가 부족한 항목을 임의 추정으로 완료할 위험이 있음

## 4. As-Is Workflow

자료 검색 → 주소·소유관계 확인 → 공식가액 별도 조회 → Excel 계산 → 근거 재확인 → 이슈 정리 → 요청자료 작성 → 검토메모 작성

As-Is에서는 결론, 계산과 근거가 서로 다른 작업물에 존재하여 검토자가 결과의 완결성과 미확인 범위를 다시 연결해야 합니다.

## 5. To-Be Workflow

공식자료 식별 → 자산·필지·납세의무자 구조화 → 과세기초자료 검증 → 규칙엔진 재계산 → 핵심 결론·Coverage 확인 → P0/P1 이슈와 요청자료 검토 → 계산조서 Drill-down → Source Lineage 확인 → 전문가 승인

To-Be는 모든 세법 판단을 자동화하지 않습니다. 반복 가능한 데이터 처리와 계산을 표준화하고 전문가 판단이 필요한 지점을 앞쪽에 드러내는 것이 목적입니다.

## 6. Data·Automation 기반 검토

| 영역 | 적용 내용 |
|---|---|
| Data | 자산·필지·건축물·납세의무자·공식가액·출처 메타데이터 구조화 |
| Automation | 세목별 재계산, 민감도 분석, 검증상태, 이슈·요청자료 연결, 결과 출력 |
| Digital Advisory | 결론과 미해결 이슈를 업무 의사결정 순서에 맞게 제공 |
| 검증 통제 | Schema 검증, Source Lineage, Coverage, Fail-closed, 회귀테스트 |
| 전문가 검토 | 법적 판단, 과세기준일 소유관계, 실제 고지서 대사와 최종 승인 |

## 7. 검증 통제와 예외처리

- 공식 입력근거가 없으면 장부가액이나 Peer 비율로 대체하지 않습니다.
- 실제 고지서가 없으면 재계산액을 확정세액 또는 `verified_notice`로 표시하지 않습니다.
- 필수 Schema 또는 참조관계가 맞지 않으면 계산 완료 상태를 차단합니다.
- 민감도는 공식 입력값 변화의 기계적 결과로만 표시하며 미래 세액을 예측하지 않습니다.
- 다운로드 결과에도 계산상태, Source Lineage와 미해결 이슈를 함께 보존합니다.

## 8. 전문가 검토 경계

등기부, 신탁원부, 공동담보목록과 임대차 공시는 소유·권리 또는 거래구조 검토를 지원하지만 공시가격·시가표준액과 실제 세금 고지서를 대체하지 않습니다. 실제 과세구분, 법정 절사, 감면, 세부담상한, 지방자치단체 조정과 신고·납부 의사결정은 전문가 검토 범위입니다.

## 10. 한계와 향후 개선

- 실제 고지서와 과세내역서가 없어 결정세액 대사는 미완료입니다.
- 과세기준일 현재 등기·신탁 원문과 실제 과세구분 확인이 필요합니다.
- 감면, 세부담상한, 법정 절사와 지방자치단체 조정은 반영하지 않았습니다.
- 다른 자산으로 확대하려면 동일한 공식 입력근거와 검증 기준을 충족해야 합니다.
- 향후에는 실제 고지서 Reconciliation, Evidence 승인 Workflow와 검토자 Sign-off 이력을 강화할 수 있습니다.
