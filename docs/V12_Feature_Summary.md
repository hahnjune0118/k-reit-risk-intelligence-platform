# v12 Feature Summary

> 참고: 현재 활성 버전은 v13 - Tax Review Pack Generator입니다. 이 문서는 v12에서 도입된 Peer Benchmark 및 Red Flag Engine의 이력을 설명합니다.

## v12 - Peer Benchmark & Red Flag Engine

v12는 상장리츠 전체를 비교대상군으로 설정하고, 선택한 리츠 회사의 감사위험 및 보유세 부담이 Peer 대비 어느 영역에서 높게 나타나는지 자동으로 식별하는 버전입니다.

## 1. Peer Benchmark

Peer Benchmark는 `data/reit_peer_snapshot.csv`를 기준으로 계산합니다.

리츠를 분석 대상으로 선택한 이유는 공개자료만으로도 자산, 임대수익, 차입금, 공정가치 평가, 보유세 부담 등 감사·세무 리스크 분석에 필요한 핵심 정보를 구조화할 수 있기 때문입니다.

주요 비교 지표:

- 총자산
- 투자부동산 / 총자산
- 부채 / 총자산
- 단기성 차입금 / 총차입금
- 이자비용 / FFO
- 배당 / FFO
- 보유세 / FFO
- 보유세 / 영업수익
- 공시가격 / 투자부동산 장부금액
- 영업현금흐름 / 배당

각 지표는 Peer Group 내 상대 위치를 함께 보여 주며, Missing Data는 무리하게 위험으로 판정하지 않고 별도 상태로 처리합니다.

## 2. 사이드바 회사 선택 workflow

v12는 공개 리뷰어가 별도 종목코드나 공시 기준일을 입력하지 않아도 분석할 수 있도록 사이드바 workflow를 단순화했습니다.

- 분석 모드를 먼저 선택
- 시가총액 순위 Snapshot 기준으로 정렬된 분석 대상회사를 선택
- 선택 회사의 종목코드와 DART corp_code 자동 연결
- 최근 5년 재무 흐름은 Snapshot을 우선 사용
- Snapshot이 부족한 경우에만 선택 회사 DART 자료를 보조적으로 사용
- 동일한 선택 회사 context를 General, Assurance, Tax 화면에서 공유

## 3. 최근 5년 흐름 표시 개선

기존처럼 금리, NAV, FFO를 하나의 축에 지수화하여 표시하면 단위가 다른 지표를 같은 의미처럼 오해할 수 있습니다. v12는 `최근 5년 흐름: 금리와 리츠 주요 지표` 표에서 기준금리, 국고채, 회사채 금리는 실제 이자율(%)로, NAV, FFO, 총자산, 차입금, 이자비용은 실제 금액(억원)으로 구분합니다.

## 4. Assurance Red Flag Engine

Assurance Red Flag Engine은 감사계획 단계에서 중점 검토 영역을 빠르게 식별하기 위한 기능입니다.

검토 관점:

- 투자부동산 공정가치 평가위험
- 차입금 만기 및 차환 위험
- 이자비용 부담
- 배당 지속가능성
- 영업현금흐름 배당 커버리지
- RMM(중요왜곡표시위험)과 KAM(핵심감사사항) 후보 검토 신호

각 Red Flag는 감사인이 다음 단계에서 수행할 수 있는 감사절차와 요청자료 목록으로 연결됩니다.

v12 UI에서는 Red Flag 카드와 요청자료 목록을 중복 표시하지 않고, `감사절차 및 요청자료` 표 하나로 통합했습니다.

## 5. Tax Red Flag Engine

Tax Red Flag Engine은 리츠 보유자산의 보유세 부담이 Peer 대비 높은지, 또는 공시가격 관련 검토가 필요한지 파악하기 위한 기능입니다.

검토 관점:

- 보유세 / FFO
- 보유세 / 영업수익
- 공시가격 / 투자부동산 장부금액
- 공시가격 상승 시 보유세 민감도
- 재산세 고지서, 공시지가, 건축물대장 등 요청자료 필요성

본 기능은 신고 목적의 세액 산출이 아니라, 보유세 부담의 방향성과 민감도를 파악하기 위한 예비 분석입니다.

v12 UI에서는 Tax Red Flag 카드와 검토 포인트 목록을 중복 표시하지 않고, `Tax 검토사항 및 요청자료` 표 하나로 통합했습니다.

## 6. Snapshot 기반 설계

공개 앱이 매번 모든 상장리츠의 DART API를 호출하면 속도와 안정성이 떨어질 수 있습니다. v12는 공개 리뷰 환경에서 안정적으로 실행되도록 Peer Snapshot을 기본 입력값으로 사용합니다.

Snapshot 갱신은 선택적 스크립트로 분리했습니다.

```powershell
py scripts\refresh_reit_peer_snapshot.py
```

이 스크립트는 앱 시작 시 자동 실행되지 않으며, 실패하더라도 기존 snapshot을 삭제하지 않는 방향으로 설계했습니다.

## 7. 공개 런타임에서 비활성화된 기능

v12 공개 버전에서는 다음 기능을 사용하지 않습니다.

- Deals mode
- KRX API
- KRX 기반 시장가치 분석
- 시장가격 기반 P/NAV, P/FFO 분석

향후 안정적인 데이터 연결과 별도 UI가 정리되면 독립 모듈로 재검토할 수 있습니다.

## 8. 한계

- Snapshot 데이터는 공개 포트폴리오 검토용 예시 데이터일 수 있습니다.
- Red Flag는 전문가 판단을 대체하지 않습니다.
- 실제 감사계획, 세무신고, 투자판단에는 원자료 확인과 전문가 검토가 필요합니다.
- Peer 비교 결과는 입력 데이터의 범위와 품질에 영향을 받습니다.
